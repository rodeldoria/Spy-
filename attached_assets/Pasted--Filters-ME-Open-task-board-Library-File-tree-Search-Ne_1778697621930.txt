App is running cleanly. Here's the summary of all three changes:

1. Concrete dollar payouts on every market row

New sidebar input — Stake size ($) (default $10). Every YES and NO column now shows a 💵 line in plain English:

💵 Bet $10 → +$0.11 if YES wins · -$10.00 if NO
💵 Bet $10 → +$990.00 if NO wins · -$10.00 if YES

The math is exact: profit-if-win = stake × (payout − 1), loss-if-lose = stake. So when you see "YES @ 1¢ (100x)", the dollar line confirms a $10 NO bet wins $990 if BTC actually fails to hit the strike. This makes the asymmetric risk/reward immediately visceral instead of buried in cents.

2. Killed the flash pulse — silent text updates only

Removed the green flash animation entirely. Spot prices now update silently — the digits just change in place, exactly like a Bloomberg or Robinhood quote.

3. Authentic live feed wired in: Coinbase WebSocket

Replaced the polling-based spot updates with a direct browser-to-Coinbase WebSocket connection at wss://ws-feed.exchange.coinbase.com:

Free, no auth, no API key needed
Subscribes to the ticker channel for every symbol on the page (BTC-USD / ETH-USD / SOL-USD)
Delivers ~5–15 ticks/second on BTC during active hours
Updates every .kalshi-spot-live span as messages arrive
Auto-reconnects after 3s if dropped
Cleans up the prior socket on each Streamlit rerun (no leaks)
This is the same feed Coinbase Advanced Trade and many pro tools use. Independent of Streamlit's auto-refresh — even if you set the slider to 5 minutes, spot prices keep flowing in real time.
